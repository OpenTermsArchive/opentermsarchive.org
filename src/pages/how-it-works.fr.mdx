## 1. Déclaration de service et document

Pour ajouter le suivi d'un document vous pouvez utiliser <Link href="/contribute">l'interface d'ajout de document</Link> sur le présent site web. C'est une interface simple qui vous permet en quelque clics de faire une demande d'ajout de document à suivre.

Pour les personnes à l'aise avec GitHub, vous pouvez soumettre une "pull request" sur le dépôt GitHub [OpenTermsArchive](https://github.com/ambanum/OpenTermsArchive) en déclarant un nouveau fichier <abbr title="JavaScript Object Notation">JSON</abbr> contenant les informations du service et du document à suivre dans le dossier `services`.

**Quelque soit la méthode envisagée, [la documentation complète](https://github.com/ambanum/OpenTermsArchive/blob/master/CONTRIBUTING.md) au sujet de la contribution pourrait vous être utile.**

La liste des services et des document suivis est [consultable ici](https://github.com/ambanum/OpenTermsArchive/tree/master/services).

## 2. Enregistrement des données brutes

Pour scruter les modifications des documents au plus près, Open Term Archive les télécharge à fréquence régulière depuis leurs adresse URL. **Chaque version des documents téléchargée** (appelée `snapshot`)** est enregistrée dans un dépôt Git** accessible ici : [OpenTermsArchive-Snapshots](https://github.com/ambanum/OpenTermsArchive-snapshots).

Ce dépôt peut être considéré comme une base de donnée. Elle est distribuée sous licence [ODC-BY 1.0](https://opendatacommons.org/licenses/by/1-0/), ce qui veut dire que vous êtes libre de la partager (de la copier, distribuer et de l'utiliser), de l'adapter (de la modifier, transformer et l'exploiter) pour autant que vous l'attribuiez au bureau de l'Ambassadeur français pour le numérique.

## 3. Enregistrement des données signifiantes

Les pages web téléchargées peuvent contenir des élements de navigation ( menu principal, fil d'ariane, liens de bas de page... ), des publicités, un champs de recherche... Ce sont des élements de la page qui sont non signifiants, appelées "bruit", c'est à dire qu'ils ne concernent pas directement le contenu qui est intéressant. **Open Terms Archive enregistre une version de ce document dans laquelle le bruit est supprimé.**

Pour ce faire, dans la déclaration [d'un nouveau service](https://github.com/ambanum/OpenTermsArchive/blob/master/CONTRIBUTING.md#tracking-new-documents) la proriété `select` permet de cibler un noeud <abbr title="Document Object Model">DOM</abbr> de la page et la propriété `remove` permet d'en supprimer. Il existe aussi un [mécanisme de filtres](https://github.com/ambanum/OpenTermsArchive/blob/master/CONTRIBUTING.md#filter) (en Javascript), qui permet pour les documents plus complexe, de faire des opérations sur le <abbr title="Document Object Model">DOM</abbr> avant de l'enregistrer.

Les données sont disponibles dans le dépôt GitHub [OpenTermsArchive-versions](https://github.com/ambanum/OpenTermsArchive-versions) et elles aussi distribuées sous licence [ODC-BY 1.0](https://opendatacommons.org/licenses/by/1-0/).

## 4. Naviguer dans l'historique des versions

Il est possible de naviguer dans l'historique des différentes version à travers l'interface GitHub du dépôt [OpenTermsArchive-versions](https://github.com/ambanum/OpenTermsArchive-versions/). Pour cela, cliquez sur le dossier du service de votre choix puis sur le type de document. Vous aurez la possibilité de voir l'ensemble des versions en cliquant sur le lien "History".

## 5. Être notifié

Toutes personnes qui souhaiteraient suivre des modifications de documents peuvent choisir d’en être notifiés par mail.

Attention toutefois, ce service est en version bêta et vous risquez de recevoir de nombreuses notifications ! Il reste possible de se désabonner en répondant à n’importe quel courriel reçu. Chaque utilisateur peut également être informé des modifications survenues sur un document contractuel en s’abonnant au flux RSS relatif à une page spécifique.

Pour en savoir plus, [c'est ici](https://github.com/ambanum/OpenTermsArchive#be-notified).
